Core Capabilities

Engineered for
serious builds.

Edge AI on CM5
Run TensorFlow Lite, PyTorch, and OpenCV directly on the Raspberry Pi CM5 Quad-core A76 at 2.4 GHz. Two MIPI CSI camera ports for stereo vision, object detection, and real-time inference at the aircraft — not the cloud.
2× MIPI CSI · HDMI Out
FMUv6 Full Compatibility
Drop-in compatible with the entire Pixhawk 6X ecosystem. JST-GH connectors throughout, PX4 and ArduPilot native support, DroneCAN, SBUS, PPM, DSM Spektrum — no rewiring, no relearning.
PX4 · ArduPilot · DroneCAN
4G/5G Built-in
M.2 modem card connected via PCIe + USB to the CM5. Nano SIM slot on-board. Live video over cellular, cloud telemetry, remote commands, and OTA updates — BVLOS ready without any external modem.
PCIe M.2 · NanoSIM · BVLOS
Triple Power Redundancy
LTC4417 priority power selector automatically fails over between three independent inputs — two 5V/4A bricks and one 12V/3A source. BQ24315 overcurrent + overvoltage protection on every downstream rail.
LTC4417 · BQ24315 · No SPF
Gigabit Ethernet
LAN8742A PHY wired directly to the FMU via RMII. MAVLink over Ethernet, high-bandwidth sensor data, plus a second Gigabit port on CM5 for network switching or ground uplink. Magnetic isolation on all lines.
LAN8742A · RMII · Isolated
16 PWM Channels
8 MAIN outputs via FMU and 8 AUX via the dedicated STM32F103 IO coprocessor. Run octocopters, VTOL hybrids, multi-payload rigs, or custom actuator configurations without hitting a channel limit.
8 MAIN + 8 AUX · DSHOT Ready

    Enclosure & Thermal
    Machined aluminium housing with ventilated top panel. Passive heatsink plus active fan header — PWM control and tachometer feedback — for sustained CM5 compute workloads at altitude.

    FMUv6 + CM5 Baseboard
    Main PCB: FMU MCU, STM32F103 IO coprocessor, LAN8742A Ethernet PHY, triple-redundant power management, and all JST-GH peripheral connectors. 150-pin DF40HC board-to-board to CM5.

    Bottom Shell + Mounting
    CNC aluminium base plate with 30.5×30.5 mm and 45×45 mm mounting hole patterns. Compatible with all professional drone frames and standard autopilot mounting configurations.

mavsdk_hbq_example.py — CM5 Ubuntu → FMU via Ethernet
# CM5 connects to FMU over local Gigabit ETH — no USB cable needed import asyncio from mavsdk import System from mavsdk.offboard import VelocityNedYaw async def run(): drone = System() # FMU reachable on ETH link between CM5 and baseboard await drone.connect(system_address="udp://192.168.10.1:14540") print("Waiting for FMU connection...") async for state in drone.core.connection_state(): if state.is_connected: break # Your ROS 2 node / OpenCV result → offboard velocity command await drone.offboard.set_velocity_ned(VelocityNedYaw(0.0, 0.0, -1.0, 0.0)) await drone.offboard.start() # 4G/5G modem streams telemetry to cloud in parallel on CM5 asyncio.run(run())
